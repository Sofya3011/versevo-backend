2 Проектирование приложения для чтения и анализа документов

Разрабатываемая система «VERSEVO» предназначена для комплексного информационно-аналитического обеспечения процессов работы пользователей с электронными документами, в части реализации следующих функций:
– хранение и поиск документов различных форматов (TXT, PDF, DOCX, EPUB) в персонализированной библиотеке;
– интеллектуальный анализ текста с использованием AI-сервисов (определение тональности, выделение ключевых тем, генерация краткого содержания);
– AI-чат для ответов на вопросы по содержанию документа;
– автоматический перевод текста между русским и английским языками с использованием моделей Hugging Face;
– аннотирование документов (создание заметок, закладок, выделение цитат с цветовой маркировкой) с привязкой к конкретным позициям в тексте;
– персонализированные настройки чтения (размер шрифта 12–28pt, 5 гарнитур шрифта, светлая/тёмная/сепия темы);
– синхронизация пользовательских данных (документы, заметки, закладки, цитаты, прогресс чтения) между устройствами через облачное хранилище.

Учитывая вышеперечисленные функции, нуждающиеся в автоматизации, следует разработать архитектуру автоматизированной системы, а также спроектировать реляционную базу данных.

2.2 Проектирование архитектуры разрабатываемого приложения

Для разработки архитектуры системы необходимо составить её структурную схему. Архитектура системы «VERSEVO» построена по принципу «клиент-сервер» и включает следующие компоненты.

Мобильный клиент (Flutter/Dart):
– пользовательский интерфейс на базе Material Design 3;
– слой управления состоянием на основе паттерна BLoC;
– API-слой для взаимодействия с сервером через HTTP-клиент Dio;
– локальное кэширование данных с использованием SharedPreferences;
– модуль прямой интеграции с Hugging Face Inference API для AI-перевода и AI-чата.

Серверная часть (Python/FastAPI):
– REST API для управления документами, пользователями, заметками и цитатами;
– модуль извлечения текста из файлов (PyMuPDF для PDF, python-docx для DOCX);
– модуль определения структуры документа (разбиение на главы на основе регулярных выражений);
– модуль определения языка текста (на основе соотношения кириллицы и латиницы);
– модуль интеграции с AI-моделями Hugging Face (перевод Helsinki-NLP, тональность rubert, NER wikineural).

Внешние AI-сервисы:
– Hugging Face Inference API – перевод (NLLB-200, Helsinki-NLP OPUS-MT), генерация ответов в AI-чате (flan-t5-base, flan-t5-small).

2.3 Проектирование структуры базы данных для приложения

PostgreSQL является одной из наиболее популярных систем управления базами данных с открытым исходным кодом в мире. Данная СУБД подходит для самых различных проектов: от небольших приложений до больших высоконагруженных проектов, включая облачные решения.

PostgreSQL характеризуется такими особенностями как:
– производительность – PostgreSQL оптимизирован для работы с большими объёмами данных и сложными запросами, поддерживает индексацию, параллельные запросы и эффективное кэширование;
– надёжность и безопасность – PostgreSQL предоставляет шифрование данных на уровне столбцов и дисков, поддерживает SSL-соединения, ролевую модель доступа и журналирование операций для восстановления после сбоев;
– масштабируемость – PostgreSQL поддерживает репликацию, шардирование (через расширения) и горизонтальное масштабирование, что особенно важно для облачных развёртываний;
– поддержка сложных типов данных – PostgreSQL изначально поддерживает JSON, JSONB, массивы, геопространственные типы (PostGIS), полнотекстовый поиск;
– совместимость и экосистема – PostgreSQL совместим с большинством современных фреймворков (включая FastAPI) и обладает богатой экосистемой расширений.

В ходе проектирования базы данных для системы интеллектуальной работы с документами «VERSEVO», учитывая все приведённые выше преимущества, было принято решение об использовании системы управления базами данных PostgreSQL в качестве основной СУБД для реализации БД системы. Дополнительным фактором выбора стала платформа Railway, на которой развёрнут бэкенд-сервер и которая предоставляет готовый managed-сервис PostgreSQL с автоматическим резервным копированием и масштабированием.

Сущности, отображённые на схеме, послужат прототипом для создания самой базы данных, её таблиц и связей между ними.
