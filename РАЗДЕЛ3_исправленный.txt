3 Реализация приложения для чтения и анализа документов

3.1 Обоснование выбора технологий разработки

Разработка мобильного приложения VERSEVO выполнялась с использованием современного, производительного и полнофункционального стека технологий, обеспечивающего быструю разработку, надёжность и превосходный пользовательский опыт.

В качестве основного языка программирования для реализации клиентской части был выбран Dart в силу его производительности, асинхронной модели и современного синтаксиса. Для создания интерфейса используется фреймворк Flutter, который обеспечивает нативную производительность на iOS и Android при единой кодовой базе, а также богатую библиотеку готовых виджетов и возможность горячей перезагрузки.

Flutter – это открытый и бесплатный фреймворк для создания кроссплатформенных приложений от компании Google. Он использует язык Dart и предоставляет разработчикам библиотеки виджетов Material Design и Cupertino, а также мощные инструменты для создания кастомных интерфейсов.

Для HTTP-запросов используется библиотека Dio, предоставляющая интерцепторы для автоматической авторизации, таймауты, повторные попытки при ошибках (до 2 повторов с экспоненциальной задержкой) и автоматическую сериализацию JSON.

Для структурирования проекта и управления состоянием используется паттерн BLoC (Business Logic Component), который разделяет бизнес-логику и представление, обеспечивая предсказуемость и тестируемость приложения. BLoC позволяет централизованно управлять состоянием экранов и обрабатывать асинхронные события (запросы к API, загрузка файлов).

В качестве серверной части выбран FastAPI – современный, быстрый веб-фреймворк для Python, который предоставляет автоматическую документацию API (Swagger UI) и высокую производительность за счёт асинхронной обработки запросов. FastAPI основан на стандартах OpenAPI и JSON Schema, что упрощает интеграцию с клиентскими приложениями.

Для базы данных используется PostgreSQL, развёрнутая на облачной платформе Railway, что обеспечивает надёжность, автоматические бэкапы и масштабируемость.

Для интеграции с AI-сервисами используется Hugging Face Inference API – облачный сервис, предоставляющий доступ к предобученным моделям. В приложении используются следующие модели:
– facebook/nllb-200-distilled-600M – перевод между 200+ языками;
– google/flan-t5-base, google/flan-t5-small – генерация ответов в AI-чате;
– blanchefort/rubert-base-cased-sentiment – анализ тональности русского текста;
– Babelscape/wikineural-multilingual-ner – извлечение именованных сущностей (люди, организации, локации).

Для локального хранения настроек пользователя (тема, размер шрифта, токен авторизации) используется SharedPreferences.

Для извлечения текста из загруженных документов на стороне сервера применяются:
– PyMuPDF (fitz) – для PDF-файлов;
– python-docx – для DOCX-файлов;
– встроенное чтение – для TXT-файлов.

Представленный стек представляет собой современную, сбалансированную и мощную комбинацию для создания полноценного мобильного приложения для чтения и анализа документов.

Он позволяет:
– максимально ускорить разработку: FastAPI предоставляет автоматическую документацию и валидацию данных, а Flutter позволяет разрабатывать для двух платформ одновременно;
– обеспечить масштабируемость: PostgreSQL на Railway легко масштабируется под растущее число пользователей и документов.

Такой технологический стек обеспечивает не только быструю и эффективную разработку, но и создание качественного, надёжного продукта, готового к использованию в реальных условиях с высокими требованиями к производительности и безопасности.

3.2 Разработка интерфейса разрабатываемого приложения

Интерфейс мобильного приложения VERSEVO разработан на языке Dart с использованием фреймворка Flutter и библиотеки виджетов Material Design 3.

В ходе разработки интерфейса были созданы следующие экраны:
– LoginScreen – экран авторизации с градиентным фоном и анимацией появления;
– RegisterScreen – экран регистрации нового пользователя;
– HomeScreen – главный экран с аватаром пользователя, статистикой и навигацией;
– LibraryScreen – экран библиотеки документов с поиском и быстрыми действиями (перевод, AI-анализ, AI-чат, удаление);
– DocumentUploadScreen – экран загрузки документов с выбором файлов (PDF, DOCX, TXT, EPUB) и прогресс-баром;
– DocumentReaderScreen – полнофункциональная читалка с навигацией по главам, настройками отображения (12–28pt, 5 гарнитур шрифта, 3 темы), переводом, заметками, закладками и цитатами;
– AnalysisScreen – экран AI-анализа с вкладками «Обзор», «Сводка» и «Статистика»;
– AiChatScreen – экран интеллектуального чата по содержанию документа;
– ReportsScreen – экран отчётов и аналитики.

3.3 Разработка программного кода разрабатываемого приложения

Flutter позволяет разрабатывать приложения, используя как виджеты, так и код программной части. Виджеты используются для определения внешнего вида приложения, а язык Dart – для реализации его поведения.

Листинг 1 – Программный код, реализующий загрузку документа на сервер

```dart
Future<void> uploadDocument() async {
  FilePickerResult? result = await FilePicker.platform.pickFiles(
    type: FileType.custom,
    allowedExtensions: ['pdf', 'docx', 'doc', 'txt', 'epub'],
  );
  if (result != null) {
    final bytes = await File(result.files.single.path!).readAsBytes();
    final base64Data = base64Encode(bytes);
    await _dio.post('/api/documents/upload-base64', data: {
      'filename': result.files.single.name,
      'file_data': base64Data,
    });
  }
}
```

Для работы с данными во Flutter используется комбинация локального хранилища (SharedPreferences) для настроек и облачного API (через Dio) для синхронизации документов и заметок.

Листинг 3 – Программный код, реализующий получение списка документов

```dart
Future<List<DocumentModel>> getDocuments() async {
  final response = await _dio.get('/api/documents', options: Options(
    headers: {'Authorization': 'Bearer $_token'},
  ));
  return (response.data as List)
      .map((json) => DocumentModel.fromJson(json))
      .toList();
}
```

Листинг 4 – Программный код AI-чата с интеграцией Hugging Face

```dart
Future<String> askQuestion(String content, String question) async {
  final prompt = 'Ты полезный ассистент. Ответь на русском языке естественно.'
      '\n\nКонтекст: $content'
      '\n\nВопрос: $question\n\nОтвет:';
  final response = await _hfDio.post(
    'https://api-inference.huggingface.co/models/google/flan-t5-base',
    data: {
      'inputs': prompt,
      'parameters': {'max_new_tokens': 300, 'temperature': 0.7},
    },
  );
  return response.data[0]['generated_text'];
}
```

Листинг 5 – Программный код, реализующий управление состоянием через BLoC

```dart
class DocumentBloc extends Bloc<DocumentEvent, DocumentState> {
  DocumentBloc() : super(DocumentInitial()) {
    on<LoadDocuments>(_onLoadDocuments);
    on<DeleteDocument>(_onDeleteDocument);
  }

  Future<void> _onLoadDocuments(
    LoadDocuments event, Emitter<DocumentState> emit) async {
    emit(DocumentLoading());
    try {
      final documents = await _documentApi.getDocuments();
      emit(DocumentLoaded(documents));
    } catch (e) {
      emit(DocumentError(e.toString()));
    }
  }
}
```
